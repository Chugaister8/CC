# Глосарій модуля 06

Усі ключові терміни модуля «Веб-безпека: OWASP Top 10 і не тільки» в одному місці. Українські терміни — за українською абеткою; англійські — в блоці A-Z.

## В

**Відображення загроз (Threat Modeling)** — структурований аналіз того, що може піти не так у системі на етапі проєктування; методології: STRIDE, PASTA, DREAD. *(6.6)*

**Вектор атаки (Attack Vector)** — шлях, яким зловмисник може атакувати систему (мережа, браузер, API, форми). *(6.1–6.11)*

## Д

**Директива (Directive)** у контексті CSP — окреме правило в Content-Security-Policy заголовку, що обмежує певний тип ресурсів (`script-src`, `img-src` тощо). *(6.1, 6.10)*

## З

**Захист у глибину для вебу** — поєднання незалежних шарів: WAF, Security Headers, input validation, output encoding, CSP. *(6.1)*

**Знімок безпеки (Security Baseline)** — мінімально прийнятний рівень захисту (заголовки, TLS, аутентифікація), від якого відштовхуються при оцінці. *(6.10, 6.13)*

## І

**Ін'єкція (Injection)** — клас вразливостей, де дані користувача інтерпретуються як команди або запити мовою, відмінною від мови застосунку. *(6.5)*

## К

**Картка загроз (Threat Card)** — short-form документ, що описує загрозу: назва, умова, наслідок, контрзахід. Виходить з Threat Modeling. *(6.6)*

**Клікджекінг (Clickjacking)** — атака, де прозорий iframe приховує справжній елемент під обманним, змушуючи користувача клікнути не туди. *(6.9)*

## М

**Межа довіри (Trust Boundary)** — точка в архітектурі системи, де рівень довіри до даних або ідентичності змінюється; потребує перевірки. *(6.6)*

**Міжсайтова підробка запиту (CSRF)** → Cross-Site Request Forgery. *(6.9)*

**Міжсайтові сценарії (XSS)** → Cross-Site Scripting. *(6.8)*

## О

**Отруєння кешу (Cache Poisoning)** — атака, де зловмисник змушує кеш зберегти шкідливу відповідь, що буде видана іншим користувачам. *(6.7)*

## П

**Підробка запиту на стороні сервера (SSRF)** → Server-Side Request Forgery. *(6.7)*

**Поверхня атаки (Attack Surface)** — сукупність точок, через які зловмисник може взаємодіяти з системою (ендпоінти, форми, файли, API). *(6.1)*

## С

**Санітизація (Sanitization)** — очищення вхідних даних від потенційно небезпечних елементів; наприклад, DOMPurify для HTML. Відрізняється від валідації (перевірки формату). *(6.8)*

**Серіалізація/Десеріалізація** — перетворення об'єктів у потік байт і назад; небезпечна десеріалізація з ненадійних джерел може призвести до RCE. *(6.7)*

**Стартова сторінка (Landing Page) Shadow API** → Improper Inventory Management. *(6.11)*

## Ф

**Фронтенд-захист** — перевірки і фільтрація на клієнтській стороні (JavaScript, HTML); NE є заміною серверного захисту — легко обходяться. *(6.1)*

## A-Z (англійські абревіатури та назви)

**ASVS (Application Security Verification Standard)** — стандарт OWASP для верифікації безпеки застосунків; три рівні (L1/L2/L3) з ~210 вимогами. *(6.2)*

**BFLA (Broken Function Level Authorization)** — доступ до функцій/endpoints, на які поточна роль не має прав. *(6.3, 6.11)*

**BOLA (Broken Object Level Authorization)** — доступ до чужих об'єктів через id в API-запиті; API-аналог IDOR. *(6.11)*

**CAPTCHA** — тест для розрізнення людей і ботів; захист від rate-based атак і abuse Business Logic. *(6.7, 6.11)*

**CORS (Cross-Origin Resource Sharing)** — механізм для контрольованого послаблення Same-Origin Policy через HTTP-заголовки. *(6.1)*

**CSP (Content Security Policy)** — HTTP-заголовок, що визначає, з яких джерел браузер може завантажувати ресурси; другий рубіж захисту від XSS. *(6.1, 6.8, 6.10)*

**CSRF (Cross-Site Request Forgery)** — атака, де зловмисний сайт примушує браузер надіслати небажаний запит від імені аутентифікованого користувача. *(6.9)*

**CVSS (Common Vulnerability Scoring System)** — стандартна шкала оцінки критичності вразливостей (0.0–10.0); Critical ≥ 9.0. *(6.2)*

**CWE (Common Weakness Enumeration)** — таксономія класів програмних слабкостей від MITRE. *(6.2)*

**DAST (Dynamic Application Security Testing)** — тестування безпеки запущеного застосунку (OWASP ZAP, Burp Suite). *(6.2)*

**DOM XSS** — різновид XSS, де payload обробляється JavaScript на клієнті без відправки на сервер. *(6.8)*

**DOMPurify** — JavaScript-бібліотека для санітизації HTML; видаляє небезпечні теги й атрибути. *(6.8)*

**HSTS (HTTP Strict Transport Security)** — заголовок, що примушує браузер завжди використовувати HTTPS для домену. *(6.4, 6.10)*

**IDOR (Insecure Direct Object Reference)** — тип Broken Access Control: доступ до об'єктів через передбачуваний ідентифікатор без перевірки прав. *(6.3)*

**Mass Assignment** — вразливість, де API приймає всі поля з запиту без whitelist; дозволяє змінити захищені поля (is_admin, role). *(6.3, 6.11)*

**MIME Sniffing** — поведінка браузера, де він «вгадує» тип файлу, ігноруючи Content-Type; може виконати JS у текстовому файлі. *(6.10)*

**Nonce (Number Used Once)** — унікальне випадкове значення, що генерується для кожного запиту; у CSP прив'язує дозволені скрипти до конкретного завантаження. *(6.8, 6.10)*

**OpenAPI / Swagger** — машиночитаний формат опису REST API; використовується для документації, генерації коду і автоматичного тестування. *(6.11)*

**OWASP (Open Web Application Security Project)** — некомерційна організація, що публікує Top 10, ASVS, Testing Guide та інші ресурси веббезпеки. *(6.2)*

**Path Traversal (Directory Traversal)** — атака, де `../` в параметрі шляху дозволяє вийти за межі дозволеної директорії. *(6.3)*

**Reflected XSS** — різновид XSS, де payload є частиною запиту і відображається у відповіді без збереження. *(6.8)*

**SAST (Static Application Security Testing)** — аналіз вихідного коду без запуску застосунку (Semgrep, Bandit, SonarQube). *(6.2)*

**Same-Origin Policy (SOP)** — браузерний механізм безпеки: скрипт одного origin не може читати стан або вміст іншого. *(6.1)*

**SameSite** — атрибут cookie, що контролює, в яких cross-site запитах cookie надсилається; захист від CSRF. *(6.1, 6.9)*

**SBOM (Software Bill of Materials)** — машиночитаний список компонентів ПЗ з версіями; стандарти CycloneDX, SPDX. *(6.7)*

**SCA (Software Composition Analysis)** — автоматизований аналіз залежностей на відомі CVE (pip-audit, npm audit). *(6.7)*

**Shadow API** — API-ендпоінти, що існують в production але не задокументовані (стара версія, debug endpoint). *(6.11)*

**SQL Injection (SQLi)** — різновид Injection, де вхідні дані вставляються в SQL-запит без параметризації; дозволяє читання/зміну/видалення даних. *(6.5)*

**SSRF (Server-Side Request Forgery)** — вразливість, де сервер виконує HTTP-запит до URL, контрольованого зловмисником (часто для доступу до internal ресурсів). *(6.7)*

**SSTI (Server-Side Template Injection)** — вразливість, де user input потрапляє в шаблонізатор і виконується як код. *(6.5)*

**Stored XSS** — різновид XSS, де payload зберігається в базі даних і виконується для всіх відвідувачів. *(6.8)*

**Union-based SQLi** — техніка SQL Injection, де UNION SELECT додає рядки з іншої таблиці до результату запиту. *(6.5)*

**WAF (Web Application Firewall)** — пристрій або сервіс, що фільтрує HTTP-трафік на основі правил для відомих атак (SQLi, XSS). *(модуль 02)*

**X-Content-Type-Options** — HTTP-заголовок `nosniff`; забороняє браузеру ігнорувати Content-Type. *(6.10)*

**X-Frame-Options** — HTTP-заголовок, що забороняє або обмежує вбудовування сторінки в `<iframe>`; захист від Clickjacking. *(6.9, 6.10)*

**XXE (XML External Entity Injection)** — вразливість у XML-парсері, де external entity читає файли або ініціює SSRF. *(6.5)*

**XSS (Cross-Site Scripting)** — виконання довільного JavaScript у браузері жертви через ін'єкцію в HTML/DOM. *(6.8)*
