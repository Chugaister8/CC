# 8.2. Загрози мобільним пристроям

Мобільний пристрій — це комп'ютер, що постійно з вами: у чужих мережах Wi-Fi, у публічних місцях, що перетинає кордони і зберігає ключі доступу до десятків сервісів. Загрози для мобільних пристроїв відрізняються від класичних настільних тим, що до технічних векторів додаються фізичні (втрата, крадіжка, візуальний нагляд) і мережеві (відкриті Wi-Fi). Розуміння цих загроз допоможе правильно розставити пріоритети захисту.

> 📖 Ключові терміни — у [глосарії модуля](00-glosariy.md).

## Мобільне шкідливе ПЗ

### Android: основна ціль

Android отримує непропорційно більше мобільного шкідливого ПЗ через:
- Можливість sideloading (встановлення APK поза Google Play).
- Відкритіший процес публікації в Play Store.
- Фрагментацію оновлень (старі пристрої без патчів).
- Домінування на ринку (72%+ глобально).

**Типи Android-малвару:**
- **Stalkerware / Commercial Spyware** — «шпигунські застосунки» начебто для батьківського контролю; використовуються для стеження без відома жертви. Записують дзвінки, читають SMS, відстежують геолокацію.
- **Banking Trojan** — перехоплює SMS з OTP, підміняє екрани банківських застосунків (overlay attack). Приклади: Anatsa, Cerberus, Sharkbot.
- **Premium SMS / Clicker** — підписує на платні сервіси або клікає на рекламу без відома користувача.
- **Cryptominer** — видобуває криптовалюту, розряджаючи батарею і перегріваючи пристрій.
- **Adware** — агресивна реклама; часто маскується під корисні утиліти.

### iOS: рідко, але серйозно

iOS малвар рідший і значно складніший через Code Signing і App Store review. Але існує:
- **Pegasus (NSO Group)** — комерційний spyware, що використовував zero-click вразливості iOS (без взаємодії жертви). Виявлений у 2021 через Amnesty International. Жертви: журналісти, активісти, урядовці.
- **XcodeGhost (2015)** — заражена версія Xcode для китайських розробників; шкідливий код потрапив у 4000+ застосунків App Store.
- **Masque Attack** — заміна легітимних застосунків через Enterprise certificate; виправлено в iOS 9.

## Jailbreak і Root

### iOS Jailbreak

Jailbreak — обхід обмежень Apple на виконання непідписаного коду. Відкриває можливості:
- Встановлення застосунків поза App Store (Cydia, Sileo).
- Тонке налаштування ОС.
- Встановлення шкідливого ПЗ без обмежень.

**Наслідки для безпеки:**
- Code Signing не працює → будь-який код може виконуватись.
- Sandbox порушений → застосунки можуть читати дані інших.
- Apple Pay і банківські застосунки блокують jailbroken пристрої.
- Jailbreak-інструменти самі можуть містити backdoor.

**Jailbreak detection:**
```swift
// Swift: перевірка ознак jailbreak (для банківських застосунків)
func isJailbroken() -> Bool {
    let paths = ["/Applications/Cydia.app",
                 "/private/var/lib/apt/",
                 "/usr/bin/ssh",
                 "/bin/bash"]
    for path in paths {
        if FileManager.default.fileExists(atPath: path) { return true }
    }
    // Спроба записати поза sandbox
    do {
        try "test".write(toFile: "/private/test.txt", atomically: true, encoding: .utf8)
        return true  // успіх = jailbreak
    } catch { return false }
}
```

### Android Root

Root — отримання привілеїв суперкористувача Linux. Надає повний контроль над системою, але:
- SafetyNet / Play Integrity API виявляє root → блокує Google Pay, деякі банки.
- Magisk (systemless root) намагається приховати root від виявлення.
- Custom ROM без сертифікації Google — втрата доступу до Play Store.
- Один шкідливий застосунок з root = повна компрометація пристрою.

## MITM-атаки через Wi-Fi

### Evil Twin (Rogue Access Point)

Зловмисник розгортає точку доступу з тим самим SSID, що і легітимна мережа (наприклад, `Kyiv_Free_WiFi`). Пристрій підключається автоматично → весь трафік проходить через зловмисника.

```
Жертва → [Evil Twin AP] → [Зловмисник's Router] → Інтернет
                 ↓
         Перехоплення HTTP трафіку,
         SSL Stripping для HTTPS,
         Injection шкідливого JS
```

**Захист:** завжди використовувати VPN у публічних мережах; вимикати автоматичне підключення до відкритих Wi-Fi.

### SSL Stripping

Навіть при HTTPS: зловмисник у позиції MITM може:
1. Перехопити початковий HTTP-запит жертви.
2. Підключитись до сервера по HTTPS від свого імені.
3. Повертати жертві HTTP → жертва не бачить HTTPS.

**Захист:** HSTS Preload (браузер завжди використовує HTTPS незалежно від запиту).

## SS7 і телефонні вразливості

**SS7 (Signaling System 7)** — протокол 1975 року, що досі управляє маршрутизацією дзвінків і SMS між операторами. Містить фундаментальні вразливості, що дозволяють:
- Перехоплювати SMS (OTP, банківські коди).
- Відстежувати геолокацію абонента.
- Переадресовувати дзвінки.

Зловмисникам потрібен доступ до SS7-мережі (доступний через підставних або корумпованих операторів). Через SS7 здійснювались атаки на Two-Factor Authentication через SMS по всьому світу.

**Висновок:** SMS-коди для 2FA вразливі через SS7. Використовуйте TOTP-застосунки або FIDO2.

## SIM Swapping

Зловмисник переконує оператора перенести ваш номер телефону на нову SIM-картку (через соціальну інженерію або підробку документів). Результат: зловмисник отримує всі ваші SMS — включаючи OTP банків і 2FA-коди.

**Захист:**
- Встановити PIN/пароль на обліковий запис у оператора.
- Перейти з SMS-2FA на TOTP або FIDO2.
- Використовувати email (а не телефон) для відновлення акаунтів де можливо.

## Фізичні загрози

**Shoulder Surfing** — перегляд екрану через плече в публічних місцях. Захист: Privacy Screen protector (фізичний фільтр), Auto-lock.

**Evil Maid Attack на мобільний** — несанкціонований фізичний доступ до пристрою (готельний персонал, кордонний контроль). Захист: сильний PIN/пароль (не Touch ID за кордоном — може бути примусово використаний), шифрування.

**Juice Jacking** — зарядні USB-порти в аеропортах і готелях можуть передавати дані. Захист: USB Data Blocker («тільки зарядка» адаптер), Power-only cable.

## Міні-вправа

Перевірте поточні налаштування безпеки вашого смартфона:

**Android:**
```
Налаштування → Безпека → Перевірте:
- Тип блокування екрана (PIN ≥ 6 цифр або пароль)
- Google Play Protect: увімкнено і актуальне
- Дозволи встановлення з невідомих джерел: вимкнено
- Android version і Security patch level (бажано не старше 3 місяців)
```

**iOS:**
```
Налаштування → Face/Touch ID і код-пароль:
- Код: ≥ 6 цифр або буквено-цифровий
- Дані після 10 спроб: Enabled
Налаштування → Конфіденційність → Моніторинг застосунків:
- Перевірте "Аналітика і покращення" → дані аналітики
```
