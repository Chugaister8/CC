# Глосарій модуля 09

Усі ключові терміни модуля «Хмарна безпека» за алфавітом. Українські терміни — за українською абеткою; англійські — в блоці A-Z.

## В

**Витік даних (Data Breach)** — несанкціонований доступ або розкриття конфіденційних даних; у хмарному контексті часто через мисконфігурацію (відкритий S3-бакет). *(9.4, 9.9)*

## Г

**Гіпервізор (Hypervisor)** — ПЗ, що дозволяє запускати кілька віртуальних машин на одному фізичному сервері; завжди захищається хмарним провайдером в IaaS-моделі. *(9.1)*

## Д

**Дрейф конфігурації (Configuration Drift)** — поступова зміна початково безпечної конфігурації через ручні правки і відсутність контролю версій інфраструктури. *(9.1, 9.7)*

## З

**Захист даних у спокої (Encryption at Rest)** — шифрування даних, що зберігаються на диску, в базі даних або сховищі. *(9.4)*

**Захист даних у русі (Encryption in Transit)** — шифрування даних під час передачі мережею (TLS, mTLS). *(9.4)*

## К

**Контейнер (Container)** — ізольований (частково) процес, що ділить ядро ОС хоста; основна одиниця розгортання в Docker/Kubernetes. *(9.5)*

**Конфігурація як код (Infrastructure as Code, IaC)** — управління хмарною інфраструктурою через декларативні файли (Terraform, CloudFormation), а не ручні дії. *(9.6)*

## М

**Мікросегментація** — поділ мережі на ізольовані сегменти з явним дозволом трафіку; у Kubernetes реалізується через Network Policies. *(9.5)*

**Модель спільної відповідальності (Shared Responsibility Model)** — розподіл обов'язків безпеки між хмарним провайдером і клієнтом залежно від моделі послуги. *(9.1)*

## П

**Принцип найменших привілеїв (Least Privilege)** — суб'єкт отримує лише мінімально необхідні права; ключовий принцип хмарного IAM. *(9.3)*

**Привілейована ескалація (Privilege Escalation)** — отримання вищих прав, ніж початково надано; у хмарі часто через ланцюжки IAM-дозволів. *(9.9)*

## Р

**Робоче навантаження (Workload)** — будь-який обчислювальний процес у хмарі: VM, контейнер, serverless функція. *(9.7)*

## С

**Серверless (Serverless / FaaS)** — модель виконання коду без управління серверами; AWS Lambda, Azure Functions. *(9.1)*

**Сегментація мережі (Network Segmentation)** — поділ мережі на ізольовані частини для обмеження lateral movement. *(9.2)*

## Х

**Хмара публічна (Public Cloud)** — мультитенантна інфраструктура, спільна для багатьох клієнтів; AWS, Azure, GCP. *(9.1)*

**Хмара приватна (Private Cloud)** — виділена інфраструктура для однієї організації. *(9.1)*

**Хмара гібридна (Hybrid Cloud)** — поєднання private і public cloud з інтеграцією. *(9.1)*

## A-Z (англійські абревіатури та назви)

**CIEM (Cloud Infrastructure Entitlement Management)** — аналіз і управління надмірними IAM-правами та entitlements у хмарі. *(9.7)*

**CIS Benchmark** — набір рекомендованих конфігурацій безпеки від Center for Internet Security для кожного хмарного провайдера. *(9.7)*

**CMK (Customer Managed Key)** — KMS-ключ, повністю контрольований клієнтом (ротація, policy, видалення). *(9.4)*

**CNAPP (Cloud-Native Application Protection Platform)** — платформа, що об'єднує CSPM, CWPP, CIEM, DSPM. *(9.7)*

**CSPM (Cloud Security Posture Management)** — автоматизоване виявлення мисконфігурацій хмарної інфраструктури. *(9.7)*

**CWPP (Cloud Workload Protection Platform)** — захист робочих навантажень під час виконання (runtime). *(9.7)*

**DAST (Dynamic Application Security Testing)** — тестування безпеки застосунку під час виконання, після деплою в staging. *(9.6)*

**DLP (Data Loss Prevention)** — автоматичне виявлення і захист чутливих даних; AWS Macie. *(9.4)*

**DSPM (Data Security Posture Management)** — класифікація і моніторинг безпеки даних у хмарі. *(9.7)*

**FaaS (Function as a Service)** → serverless модель виконання коду. *(9.1)*

**IaaS (Infrastructure as a Service)** — оренда обчислювальної інфраструктури; AWS EC2, Azure VM. *(9.1)*

**IAM (Identity and Access Management)** — управління ідентичностями і правами доступу в хмарі. *(9.3)*

**IMDS (Instance Metadata Service)** — внутрішній endpoint (169.254.169.254), що надає тимчасові credentials інстанції; ціль SSRF-атак. *(9.9)*

**IRSA (IAM Roles for Service Accounts)** — механізм AWS EKS для призначення IAM-ролей окремим Kubernetes Service Accounts замість всієї ноди. *(9.9)*

**KMS (Key Management Service)** — керований сервіс для генерації, зберігання і використання криптографічних ключів. *(9.4)*

**MFA Delete** — додатковий захист S3 versioning, що вимагає MFA для остаточного видалення об'єктів. *(9.4)*

**NACL (Network Access Control List)** — stateless firewall на рівні підмережі VPC. *(9.2)*

**PaaS (Platform as a Service)** — платформа для запуску застосунків без управління ОС; Google App Engine, Heroku. *(9.1)*

**Permission Boundary** — максимально допустимі IAM-дозволи незалежно від призначеної Policy. *(9.3)*

**RBAC (Role-Based Access Control)** — модель контролю доступу через ролі; в Azure і Kubernetes. *(9.3, 9.5)*

**SaaS (Software as a Service)** — готовий застосунок через браузер; Microsoft 365, Salesforce. *(9.1)*

**SAST (Static Application Security Testing)** — аналіз вихідного коду на вразливості без виконання. *(9.6)*

**SBOM (Software Bill of Materials)** — машиночитаний список компонентів застосунку. *(9.6)*

**SCA (Software Composition Analysis)** — аналіз залежностей на відомі CVE. *(9.6)*

**SCP (Service Control Policy)** — максимально допустимі права для всього AWS Organizations акаунту. *(9.3)*

**Security Group** — stateful firewall на рівні мережевого інтерфейсу AWS. *(9.2)*

**SG** → Security Group.

**Shift Left** — інтеграція перевірок безпеки на ранніх етапах розробки (IDE, code review) замість лише перед релізом. *(9.6)*

**SLSA (Supply chain Levels for Software Artifacts)** — фреймворк рівнів довіри до процесу збірки ПЗ. *(9.6)*

**SOC 2** — аудиторський стандарт для сервісних організацій, що обробляють дані клієнтів. *(9.8)*

**SRM (Shared Responsibility Model)** → модель спільної відповідальності.

**SSRF (Server-Side Request Forgery)** — змушує сервер виконати запит до внутрішнього ресурсу; у хмарі — вектор до Instance Metadata Service. *(9.9)*

**VPC (Virtual Private Cloud)** — ізольована мережева середа у хмарному провайдері. *(9.2)*

**VPC Peering** — пряме непрозоре з'єднання між двома VPC. *(9.2)*

**WAF (Web Application Firewall)** — фільтр HTTP/HTTPS трафіку на рівні L7. *(9.2)*

**Well-Architected Framework** — методологія AWS для оцінки архітектури проти найкращих практик шести стовпів. *(9.1, 9.8)*

**Workload Identity Federation** — отримання хмарних credentials зовнішніми системами без статичних ключів. *(9.3)*
