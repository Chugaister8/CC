# Глосарій модуля 11

Усі ключові терміни модуля «Цифрова криміналістика та реагування на інциденти» за алфавітом. Українські терміни — за українською абеткою; англійські — в блоці A-Z.

## А

**Атрибуція (Attribution)** — встановлення джерела або виконавця атаки на основі зібраних доказів; технічно складна і часто остаточно недосяжна без додаткових розвідданих. *(11.5)*

## В

**Волатильність (Volatility)** — швидкість, з якою цифрові дані зникають або змінюються; основа для визначення порядку збору доказів (Order of Volatility). *(11.1)*

**Витяг логічний (Logical Extraction)** — отримання даних мобільного пристрою через офіційні API без глибокого технічного обходу захисту. *(11.6)*

**Витяг фізичний (Physical Extraction)** — побітова копія flash-пам'яті мобільного пристрою; найповніший, але технічно найскладніший рівень доступу. *(11.6)*

## Г

**Гіпотезо-орієнтований підхід** — методологія форензичного аналізу через формулювання і перевірку альтернативних гіпотез, а не підтвердження заздалегідь прийнятого висновку. *(11.1)*

## Д

**Доказ цифровий (Digital Evidence)** — будь-яка інформація доказової цінності, що зберігається або передається в цифровому вигляді. *(11.1, 11.2)*

## Е

**Ефемерний ресурс (Ephemeral Resource)** — хмарний або контейнерний ресурс з коротким життєвим циклом, що ускладнює традиційний форензичний збір доказів. *(11.7)*

## З

**Захоплення пам'яті (Memory Dump/Acquisition)** — процес копіювання вмісту оперативної пам'яті системи для подальшого аналізу. *(11.4)*

## К

**Карвінг файлів (File Carving)** — техніка відновлення файлів безпосередньо з необробленого образу диска без покладання на метадані файлової системи. *(11.3)*

**Криміналістика цифрова (Digital Forensics)** — застосування наукових методів для збору, збереження, аналізу і представлення цифрових доказів. *(11.1)*

## Л

**Ланцюжок зберігання доказів (Chain of Custody)** — документована хронологія контролю, передачі і розпорядження доказами від моменту збору до представлення. *(11.2)*

## М

**МЕЙС (MACE/MACB)** — абревіатура для чотирьох типів часових міток NTFS-файлу: Modified, Accessed, Created/Changed, (Born) Entry Modified. *(11.3)*

## Н

**Незмінне сховище (Immutable Storage)** — сховище доказів, що технічно унеможливлює видалення або модифікацію навіть з адміністративним доступом протягом визначеного періоду. *(11.7, 11.8)*

## П

**Принцип обміну Локара (Locard's Exchange Principle)** — «кожен контакт залишає слід»; основоположний принцип криміналістики, застосовний і до цифрового світу. *(11.1)*

**Підробка доказів (Evidence Manipulation)** — навмисна зміна цифрових доказів для введення розслідування в оману (timestomping, log injection). *(11.8)*

## С

**Слек-простір (Slack Space)** — простір між кінцем фактичних даних файлу і кінцем виділеного йому кластера диска; може містити залишки попередніх даних. *(11.3)*

**Стеганографія (Steganography)** — приховування інформації всередині іншого, на вигляд невинного файлу. *(11.8)*

**Супер-таймлайн (Super Timeline)** — об'єднання часових міток з усіх можливих джерел доказів в єдину хронологічну послідовність. *(11.3)*

## Т

**Таймстомпінг (Timestomping)** — anti-forensics техніка зміни часових міток файлу для приховування справжнього часу дій. *(11.8)*

## Ф

**Форензична строгість (Forensic Soundness)** — якість методології, що гарантує незмінність доказів і відтворюваність результатів незалежною стороною. *(11.1, 11.2)*

---

## A-Z (англійські абревіатури та назви)

**ACPO (Association of Chief Police Officers)** — британська організація, чиї чотири принципи стали де-факто міжнародним стандартом цифрової криміналістики. *(11.1)*

**ADS (Alternate Data Streams)** — функція NTFS, що дозволяє приховувати дані в «альтернативних потоках» файлу, непомітних при звичайному перегляді. *(11.8)*

**bulk_extractor** — інструмент паралельного file carving з автоматичним виявленням артефактів (email, URL, номери карток). *(11.3)*

**Cellebrite UFED** — провідне комерційне рішення мобільної форензики, що використовується правоохоронними органами. *(11.6)*

**Chain of Custody** → ланцюжок зберігання доказів.

**EXIF (Exchangeable Image File Format)** — метадані зображення, що можуть містити геолокацію, модель камери, час зйомки. *(11.10)*

**Falco** — runtime security tool для контейнерів, що може слугувати джерелом форензичних доказів для вже знищених контейнерів. *(11.7)*

**JA3/JA3S Fingerprinting** — техніка ідентифікації клієнта/сервера за унікальними параметрами TLS ClientHello, дієва навіть для зашифрованого трафіку. *(11.5)*

**LiME (Linux Memory Extractor)** — інструмент для захоплення пам'яті Linux-систем через kernel module. *(11.4)*

**log2timeline/plaso** — індустріальний стандарт для побудови super timeline з багатьох джерел доказів. *(11.3)*

**MACB/MACE** → див. українську секцію.

**MFT (Master File Table)** — центральна структура NTFS, що містить запис метаданих для кожного файлу і директорії. *(11.3)*

**Order of Volatility** — порядок збору доказів від найбільш до найменш «летких» даних. *(11.1)*

**Prefetch** — механізм Windows, що кешує інформацію про запущені програми; цінне форензичне джерело доказів виконання. *(11.3)*

**Shimcache (AppCompatCache)** — реєстрова структура Windows, що фіксує факт існування виконуваного файлу за певним шляхом у певний час. *(11.3)*

**Slack Space** → слек-простір.

**SWGDE (Scientific Working Group on Digital Evidence)** — організація, що публікує стандарти найкращих практик цифрової криміналістики. *(11.2)*

**Timestomping** → таймстомпінг.

**USN Journal (Update Sequence Number Journal)** — журнал транзакцій NTFS, корисний для виявлення розбіжностей при timestomping. *(11.8)*

**Volatility Framework** — провідний відкритий framework для аналізу дампів оперативної пам'яті. *(11.4)*

**Write Blocker** — апаратний або програмний пристрій, що дозволяє читання з носія, але фізично блокує команди запису. *(11.2)*
