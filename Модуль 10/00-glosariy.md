# Глосарій модуля 10

Усі ключові терміни модуля «Мережева безпека» за алфавітом. Українські терміни — за українською абеткою; англійські — в блоці A-Z.

## А

**Аномальне виявлення (Anomaly-Based Detection)** — метод IDS/IPS, що порівнює поточну поведінку мережі з базовим профілем «нормального» трафіку; виявляє нові, раніше невідомі атаки на відміну від сигнатурного методу. *(10.3)*

**Аутентифікація 802.1X** — стандарт мережевої автентифікації на рівні порту; вимагає підтвердження особи перед наданням мережевого доступу. *(10.6)*

## Б

**Базова лінія (Baseline)** — профіль «нормальної» поведінки мережі (типовий обсяг трафіку, час активності, протоколи), відносно якого виявляються аномалії. *(10.10)*

## В

**Виявлення вторгнень (Intrusion Detection)** — процес моніторингу мережі або системи на ознаки шкідливої активності або порушення політик. *(10.3)*

## Г

**Гостьова мережа (Guest Network)** — ізольований сегмент Wi-Fi для відвідувачів, відокремлений від корпоративної мережі. *(10.6, 10.9)*

## Д

**Демілітаризована зона (DMZ)** — мережевий сегмент між зовнішньою і внутрішньою мережею для розміщення публічно доступних сервісів (вебсервер, поштовий сервер) з обмеженим доступом до внутрішньої мережі. *(10.5)*

## З

**Запобігання вторгнень (Intrusion Prevention)** — активне блокування виявленої шкідливої активності в реальному часі, на відміну від пасивного виявлення IDS. *(10.3)*

## К

**Контроль доступу до мережі (NAC)** — система перевірки і управління пристроями, що намагаються підключитись до мережі. *(10.6)*

## М

**Мікросегментація** — поділ мережі на максимально дрібні ізольовані сегменти (до рівня окремого workload), з явним дозволом трафіку між ними. *(10.5)*

## Н

**Наскрізне шифрування тунелю (VPN Tunnel)** — захищений логічний канал передачі даних через незахищену мережу (інтернет). *(10.4)*

## П

**Пакетна фільтрація (Packet Filtering)** — найпростіший тип firewall, що перевіряє заголовки пакетів без відстеження стану з'єднання. *(10.2)*

**Посадур-оцінка (Posture Assessment)** — перевірка стану безпеки пристрою (антивірус, патчі, шифрування) перед наданням мережевого доступу через NAC. *(10.6)*

**Проксі-сервер (Proxy Server)** — посередник між клієнтом і сервером; forward proxy приховує клієнта, reverse proxy приховує сервер. *(10.8)*

## Р

**Розширене виявлення і реагування (XDR)** — еволюція EDR, що корелює дані з мережі, кінцевих точок, хмари і пошти в єдину платформу виявлення загроз. *(10.10)*

## С

**Сигнатурне виявлення (Signature-Based Detection)** — метод IDS/IPS, що порівнює трафік з базою відомих патернів атак; швидкий і точний для відомих загроз, безсилий проти нових. *(10.3)*

**Сегментація мережі (Network Segmentation)** — поділ мережі на логічно ізольовані частини для обмеження поширення атак. *(10.5)*

**Спліт-тунелювання (Split Tunneling)** — конфігурація VPN, де лише частина трафіку (корпоративний) проходить через тунель, а решта (звичайний інтернет) — напряму. *(10.4)*

**Stateful Inspection** — другий тип firewall, що відстежує стан з'єднання (TCP handshake, послідовність пакетів) для прийняття рішень. *(10.2)*

## Т

**Тунелювання DNS (DNS Tunneling)** — техніка приховування даних усередині DNS-запитів для обходу firewall; використовується для C2-комунікації і ексфільтрації даних. *(10.7)*

## Ф

**Фаєрвол наступного покоління (NGFW)** — firewall з глибокою інспекцією пакетів (DPI), розпізнаванням застосунків, інтеграцією Threat Intelligence. *(10.2)*

---

## A-Z (англійські абревіатури та назви)

**AH (Authentication Header)** — компонент IPSec, що забезпечує автентифікацію і цілісність пакету (без шифрування вмісту). *(10.4)*

**CASB (Cloud Access Security Broker)** — проміжний сервіс контролю доступу і видимості хмарних застосунків; інтегрується з SWG. *(10.8)*

**DGA (Domain Generation Algorithm)** — алгоритм, що malware використовує для генерації великої кількості псевдовипадкових доменних імен для C2-комунікації, ускладнюючи блокування. *(10.7)*

**DLP (Data Loss Prevention)** — технологія запобігання витоку чутливих даних через мережеві канали. *(10.8)*

**DMZ** → демілітаризована зона.

**DNSSEC (DNS Security Extensions)** — розширення DNS з криптографічними підписами для захисту від підробки відповідей (DNS spoofing/cache poisoning). *(10.7)*

**DoH (DNS over HTTPS)** — шифрування DNS-запитів через HTTPS; ускладнює моніторинг DNS-трафіку організацією. *(10.7)*

**DoT (DNS over TLS)** — шифрування DNS-запитів через окремий TLS-порт (853). *(10.7)*

**ESP (Encapsulating Security Payload)** — компонент IPSec, що забезпечує шифрування і опціонально автентифікацію пакету. *(10.4)*

**HIDS (Host-based IDS)** — система виявлення вторгнень, встановлена на конкретному хості; моніторить локальні логи, файлову систему, процеси. *(10.3)*

**IKE (Internet Key Exchange)** — протокол узгодження ключів для IPSec VPN. *(10.4)*

**iptables/nftables** — інструменти налаштування firewall у Linux; nftables — сучасніша заміна iptables. *(10.2)*

**NAC (Network Access Control)** → контроль доступу до мережі.

**NetFlow** — протокол Cisco для збору метаданих про мережеві потоки (хто, з ким, скільки даних, який протокол) без захоплення вмісту пакетів. *(10.10)*

**NGFW (Next-Generation Firewall)** → фаєрвол наступного покоління.

**NIDS (Network-based IDS)** — система виявлення вторгнень, що аналізує мережевий трафік у конкретній точці мережі. *(10.3)*

**PCAP (Packet Capture)** — формат файлу для зберігання захопленого мережевого трафіку; аналізується через Wireshark/tcpdump. *(10.10)*

**RADIUS (Remote Authentication Dial-In User Service)** — протокол централізованої автентифікації, авторизації і обліку (AAA), що використовується в 802.1X. *(10.6)*

**Rogue AP (Rogue Access Point)** — несанкціонована точка доступу Wi-Fi, встановлена зловмисником або співробітником без дозволу; вектор MITM-атак. *(10.9)*

**SD-WAN (Software-Defined WAN)** — програмно-визначена технологія управління WAN-з'єднаннями між офісами/хмарою з централізованою політикою. *(10.5)*

**sFlow** — альтернатива NetFlow для збору мережевої телеметрії через семплювання пакетів. *(10.10)*

**SPAN (Switched Port Analyzer)** — функція комутатора для дублювання трафіку на окремий порт для моніторингу (mirror port). *(10.10)*

**Suricata** — відкритий NIDS/NIPS з підтримкою multi-threading і глибокою інспекцією протоколів. *(10.3)*

**SWG (Secure Web Gateway)** — шлюз, що фільтрує веб-трафік користувачів, блокуючи шкідливі сайти і застосунки. *(10.8)*

**TAP (Test Access Point)** — апаратний пристрій для пасивного захоплення мережевого трафіку без втручання в потік (на відміну від SPAN). *(10.10)*

**UTM (Unified Threat Management)** — універсальний пристрій, що поєднує firewall, IDS/IPS, VPN, антивірус і фільтрацію контенту в одному рішенні. *(10.2)*

**WireGuard** — сучасний VPN-протокол з мінімальною кодовою базою, високою швидкістю і стійкою криптографією (Curve25519, ChaCha20). *(10.4)*

**WPA3 (Wi-Fi Protected Access 3)** — поточний стандарт безпеки Wi-Fi з SAE (Simultaneous Authentication of Equals) замість вразливого 4-way handshake WPA2. *(10.9)*

**ZTNA (Zero Trust Network Access)** — архітектура мережевого доступу, що надає доступ лише до конкретного ресурсу після безперервної перевірки, замінюючи традиційний VPN. *(10.5)*
