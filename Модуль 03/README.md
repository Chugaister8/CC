# Модуль 03. ОС та hardening: захист робочої станції

> Найбільш укріплена мережа не врятує, якщо кінцевий пристрій — ноутбук співробітника або домашній ПК — налаштований за замовчуванням, без оновлень, із правами адміністратора для щоденної роботи. Саме кінцевий пристрій найчастіше стає першою точкою входу для зловмисника: через фішинговий лист, шкідливий вкладений файл або вразливість у браузері. Цей модуль перетворює операційну систему з «коробки» на свідомо налаштований, захищений робочий інструмент.

## Цілі модуля

Після опрацювання модуля ви зможете:

- **Запам'ятати:** перелічити ключові принципи hardening; назвати компоненти архітектури ОС, що мають значення для безпеки; назвати основні типи шкідливого ПЗ.
- **Зрозуміти:** пояснити, як модель прав доступу ОС обмежує шкоду від скомпрометованого процесу; описати роботу BitLocker/LUKS; пояснити різницю між AV і EDR.
- **Застосувати:** виконати повний цикл hardening Windows і Linux за чек-листом; налаштувати шифрування диска; написати Python-скрипт аудиту системи; перевірити стан бекапів.

**Орієнтовна тривалість модуля:** 10–12 годин (теорія 4–5 год, практика hardening 3–4 год, Python-лаби 1–2 год, самоперевірка 1 год).
