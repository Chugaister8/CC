# 3.8. Браузер і мережева безпека на рівні пристрою

Браузер — найбільш відкрита до зовнішнього світу частина вашого пристрою. Він постійно отримує і виконує код з невідомих джерел, відкриває мережеві з'єднання до сотень різних серверів і має доступ до файлів, паролів і сесій. Більшість успішних атак на кінцевого користувача відбуваються саме через браузер або пошту. При цьому браузер налаштовується за п'ять хвилин, а більшість з описаних налаштувань — це зміна кількох опцій у налаштуваннях і встановлення одного-двох розширень.

> 📖 Ключові терміни — у [глосарії модуля](00-glosariy.md).

## Вибір браузера

З точки зору безпеки і приватності:

| Браузер | Безпека | Приватність | Примітка |
|---|---|---|---|
| **Firefox** | Висока | Висока | Відкритий код, гнучке налаштування, не залежить від Google |
| **Chrome/Chromium** | Висока | Середня | Передовий sandbox, але залежить від Google-екосистеми |
| **Brave** | Висока | Висока | Chromium-основа, вбудований блокувальник трекерів |
| **Safari** | Висока | Середня | Лише macOS/iOS; хороший ITP (Intelligent Tracking Prevention) |
| **Edge** | Висока | Середня | Chromium-основа з додатковими Microsoft-функціями |
| **Opera, UC Browser** | Низька | Низька | Закритий код, сумнівна юрисдикція, не рекомендується |

Головне правило: використовуйте браузер від відомого, перевіреного виробника і тримайте його оновленим. Браузери — одна з найагресивніше атакованих поверхонь, і вендори регулярно закривають критичні вразливості.

## Hardening Firefox: практичне налаштування

### Важливі налаштування (about:preferences)

**Загальні:**
- Вимкніть «Відкривати файли після завантаження» — завжди запитувати, що робити з файлом.

**Пошук:**
- Змініть пошукову систему за замовчуванням на DuckDuckGo або Brave Search (менше відстеження).

**Приватність і захист (about:preferences#privacy):**
- Enhanced Tracking Protection: **Strict** (суворий режим).
- Надсилати сигнал «Do Not Track»: **Always** (хоч сайти й ігнорують, але сигналізує наміри).
- Cookies та дані сайтів: «Видаляти при закритті Firefox» (для максимальної приватності).
- HTTPS-Only Mode: **Enable** — браузер автоматично переходить на HTTPS скрізь де можливо.

**Безпека:**
- Блокувати небезпечний і оманливий вміст: увімкнено.
- Блокувати небезпечне завантаження: увімкнено.

### Розширені налаштування (about:config)

```
# Відкрити в адресному рядку: about:config

# Вимкнути WebRTC leak (може розкривати реальний IP за VPN)
media.peerconnection.enabled = false

# Вимкнути геолокацію
geo.enabled = false

# Вимкнути Telemetry
datareporting.healthreport.uploadEnabled = false
datareporting.policy.dataSubmissionEnabled = false
toolkit.telemetry.enabled = false

# Увімкнути DNS over HTTPS (DoH)
network.trr.mode = 2  # 2 = DoH з fallback; 3 = лише DoH
network.trr.uri = https://dns.cloudflare.com/dns-query
# або NextDNS: https://dns.nextdns.io/YOUR_ID

# Вимкнути JavaScript (дуже суворо — ламає більшість сайтів, але максимальний захист)
# javascript.enabled = false  # Не рекомендується для звичайного використання

# Вимкнути autoplay медіа
media.autoplay.default = 5
```

## Рекомендовані розширення для безпеки

| Розширення | Що робить | Рекомендованість |
|---|---|---|
| **uBlock Origin** | Блокує рекламу, трекери, шкідливі домени; ефективна фільтрація | Must |
| **Bitwarden / KeePassXC** | Менеджер паролів з відкритим кодом | Must |
| **Privacy Badger** (EFF) | Блокує сторонні трекери | Should |
| **HTTPS Everywhere** | Примусовий HTTPS (Firefox вже має вбудований) | Optional |
| **NoScript** | Блокує JS, Java, Flash за білим списком | Для параноїків |

**Чого НЕ варто робити:**
- Встановлювати занадто багато розширень — кожне є потенційним вектором атаки і має доступ до вмісту сторінок.
- Встановлювати розширення від незнайомих розробників з малою кількістю відгуків.
- Встановлювати розширення поза офіційними магазинами.

### Менеджери паролів: чому це критично важливо

Найслабше місце в безпеці більшості людей — повторне використання паролів. Менеджер паролів вирішує цю проблему: він генерує і зберігає унікальні, сильні паролі для кожного сайту.

**Порівняння:**

| Менеджер | Відкритий код | Хмара | Self-hosted | Рекомендація |
|---|---|---|---|---|
| **Bitwarden** | ✅ | ✅ | ✅ | Краща комбінація для більшості |
| **KeePassXC** | ✅ | ❌ (файл) | ✅ | Максимальна приватність, менша зручність |
| **1Password** | ❌ | ✅ | ❌ | Комерційний, хороший UX |
| **LastPass** | ❌ | ✅ | ❌ | Численні витоки — не рекомендується |
| **Вбудований у браузер** | Частково | ✅ | ❌ | Зручно, але прив'язано до одного браузера/екосистеми |

**Мінімальна конфігурація Bitwarden:**
1. Створити обліковий запис на bitwarden.com або встановити Vaultwarden (self-hosted).
2. Встановити розширення для браузера.
3. Увімкнути MFA (TOTP або YubiKey) для облікового запису Bitwarden.
4. Увімкнути автозаповнення лише для сайтів, що відповідають збереженому URL (захист від фішингу — якщо URL не збігається, Bitwarden не запропонує пароль).

## DNS на рівні пристрою

Налаштування захищеного DNS резолвера на рівні ОС (а не лише у браузері) захищає весь мережевий трафік пристрою.

**Windows:**
```powershell
# Налаштувати DoH на рівні Windows (Windows 11 1.0+)
# Панель управління → Мережа → Властивості адаптера → IPv4 → DNS сервер
# Або через Settings → Network → DNS over HTTPS

# Через PowerShell (Windows 11):
Set-DnsClientDohServerAddress -ServerAddress "1.1.1.1" -DohTemplate "https://cloudflare-dns.com/dns-query" -AllowFallbackToUdp $false
Set-DnsClientDohServerAddress -ServerAddress "8.8.8.8" -DohTemplate "https://dns.google/dns-query" -AllowFallbackToUdp $false
```

**Linux:**
```bash
# Systemd-resolved з DoT (DNS over TLS)
sudo nano /etc/systemd/resolved.conf
```
```ini
[Resolve]
DNS=1.1.1.1#cloudflare-dns.com 8.8.8.8#dns.google
FallbackDNS=9.9.9.9#dns.quad9.net
DNSOverTLS=yes
DNSSEC=yes
```
```bash
sudo systemctl restart systemd-resolved
# Перевірка:
resolvectl status
```

**Публічні DoH/DoT резолвери:**

| Провайдер | IPv4 | DoH URL | Фільтрація |
|---|---|---|---|
| Cloudflare | 1.1.1.1 | https://cloudflare-dns.com/dns-query | Ні (1.1.1.1); Malware (1.1.1.2) |
| Google | 8.8.8.8 | https://dns.google/dns-query | Ні |
| Quad9 | 9.9.9.9 | https://dns.quad9.net/dns-query | Так (відомі шкідливі домени) |
| NextDNS | Динамічний | Власний URL | Налаштовується |

## VPN-клієнт: конфігурація і поведінка

Детально VPN розглянуто в модулі 02. Тут — практичні налаштування для кінцевого пристрою.

**Kill Switch** — критично важлива функція VPN-клієнта: якщо VPN-з'єднання розривається, Kill Switch блокує весь інтернет-трафік до відновлення з'єднання. Без Kill Switch при розриві VPN трафік продовжується незахищеним каналом — і ви не помітите цього.

**Split tunneling** — функція, що дозволяє визначити, який трафік іде через VPN, а який — напряму. Корисно для продуктивності, але знижує захист для трафіку поза VPN.

**DNS Leak Test:** навіть з активним VPN DNS-запити можуть іти в обхід VPN-тунелю (DNS leak). Перевірте на `dnsleaktest.com` — всі DNS-сервери мають належати VPN-провайдеру або вашому DoH-резолверу.

## Захист від фішингу: технічні заходи

Технічні заходи допомагають, але жоден не замінює уважність:

- **HTTPS-only режим** браузера — відмовляється відкривати HTTP-сайти (або попереджає).
- **Браузерний Safe Browsing** (Google/Microsoft SmartScreen) — перевіряє URL по базі відомих фішингових сайтів.
- **uBlock Origin з фільтром фішингу** — блокує відомі фішингові домени.
- **Перевірка сертифіката** — перед введенням чутливих даних перевіряйте, чи сертифікат виданий правильній організації (не лише наявність HTTPS).
- **Менеджер паролів як захист** — якщо URL не збігається з збереженим, менеджер паролів не запропонує облікові дані автоматично.

## Мережеві налаштування пристрою

```powershell
# Windows: переглянути активні мережеві підключення
netstat -ano | findstr ESTABLISHED

# Вимкнути IPv6 якщо не використовується (зменшує поверхню атаки)
Set-NetAdapterBinding -Name "*" -ComponentID ms_tcpip6 -Enabled $false

# Вимкнути LLMNR через реєстр
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient" -Name "EnableMulticast" -Value 0
```

```bash
# Linux: переглянути активні з'єднання
ss -tulpn

# Відключити IPv6 якщо не потрібен (додати до sysctl.conf)
net.ipv6.conf.all.disable_ipv6 = 1
net.ipv6.conf.default.disable_ipv6 = 1
```

## Міні-вправа

1. Відкрийте `about:preferences#privacy` у Firefox і перевірте рівень захисту від трекінгу.
2. Зайдіть на `coveryourtracks.eff.org` — це покаже, наскільки ваш браузер «унікальний» для відстеження навіть без cookies.
3. Зайдіть на `dnsleaktest.com` і перевірте, чи ваш поточний DNS-резолвер — той, що ви налаштували.
4. Відкрийте список розширень браузера і видаліть усі, якими ви не користуєтесь активно.
