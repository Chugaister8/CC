# Глосарій модуля 14

Ключові терміни модуля «Безпека операційних систем (Windows і Linux hardening)». Українські терміни — за українською абеткою; акроніми та англомовні терміни — в окремому блоці A-Z.

## В

**Віртуалізована безпека (Virtualization-Based Security, VBS)** — технологія Windows, що використовує апаратну віртуалізацію (Hyper-V) для ізоляції найчутливіших секретів ОС в окремому, недосяжному для основної ОС контейнері; основа Credential Guard. *(14.4)*

## Д

**Дискреційний контроль доступу (Discretionary Access Control, DAC)** — модель прав доступу, за якої власник ресурсу на власний розсуд вирішує, кому надати доступ; класична модель прав Unix/Linux (власник/група/інші). *(14.5)*

**Дрейф конфігурації (Configuration Drift)** — поступове відхилення фактичного стану системи від задекларованого безпечного базового профілю через ручні зміни, необновлені політики чи неконтрольовані оновлення. *(14.7)*

## З

**Заповнення прогалини деталізації** → Sysmon.

## К

**Конфігурація як код (Configuration as Code)** — підхід, за якого бажаний стан системи описується декларативно у файлі, а спеціалізований інструмент автоматично приводить систему до цього стану й виправляє відхилення. *(14.7)*

## М

**Мандатний контроль доступу (Mandatory Access Control, MAC)** — модель, за якої централізована системна політика, а не власник ресурсу, примусово визначає дозволену поведінку процесу незалежно від прав користувача, від імені якого він запущений. *(14.6)*

## О

**Обов'язковий базовий мінімум** → CIS Benchmarks Level 1.

## П

**Патч-менеджмент операційних систем** — процес планового оновлення ОС на масштабі парку машин через централізовані інструменти (WSUS, unattended-upgrades), продовження загального Patch Management з Модуля 12 на рівні конкретної платформи. *(14.7)*

**Плоска адміністративна модель** — антипатерн, за якого один обліковий запис адміністратора використовується для входу на системи будь-якого рівня критичності, створюючи єдину точку компрометації для всієї інфраструктури. *(14.11)*

## Р

**Рівень цілісності (Integrity Level)** — механізм Windows, що класифікує процеси за рівнем довіри (Low/Medium/High/System), додатково обмежуючи взаємодію процесів нижчого рівня з ресурсами вищого навіть у межах прав одного користувача. *(14.2)*

## С

**Спеціальні біти прав (SUID, SGID, Sticky Bit)** — розширення класичної моделі прав Unix, що дозволяють виконуваному файлу тимчасово діяти з правами власника (SUID) чи групи (SGID) файлу, а не користувача, що його запускає; критична поверхня атаки при неправильному використанні. *(14.5)*

## Т

**Токен доступу (Access Token)** — об'єкт Windows, що прикріплюється до кожного процесу й містить ідентифікатор користувача, групи, до яких він належить, і його привілеї — основа моделі авторизації Windows. *(14.2)*

## Ц

**Централізоване управління безпекою** → Group Policy.

## A-Z (акроніми та міжнародні терміни)

**ACL (Access Control List)** — список записів, що визначають, які користувачі чи групи мають який тип доступу до конкретного ресурсу Windows. *(14.2)*

**auditd (Linux Audit Framework)** — підсистема ядра Linux для деталізованого журналювання системних викликів, доступу до файлів і виконання команд, аналог Sysmon для Linux. *(14.9)*

**BYOVD (Bring Your Own Vulnerable Driver)** — техніка обходу захисних механізмів користувацького рівня через встановлення легітимного, але вразливого підписаного драйвера для отримання доступу на рівні ядра. *(14.4)*

**Credential Guard (Windows Defender Credential Guard)** — механізм Windows на основі VBS, що ізолює NTLM-хеші та Kerberos-квитки в окремому віртуалізованому контейнері, недосяжному для класичних технік дампінгу LSASS. *(14.4)*

**DSC (PowerShell Desired State Configuration)** — декларативний інструмент Windows для опису й автоматичного підтримання бажаного стану конфігурації системи. *(14.3)*

**eBPF (extended Berkeley Packet Filter)** — механізм ядра Linux, що дозволяє безпечно виконувати верифіковані програми всередині ядра для перехоплення системних викликів, мережевих пакетів і файлових операцій без ризиків кастомного kernel-модуля. *(14.10)*

**EDR (Endpoint Detection and Response)** — клас захисних платформ, що забезпечують поведінковий моніторинг у реальному часі та можливість активної реакції на підозрілу активність, на відміну від сигнатурного антивірусу. *(14.10)*

**ETW (Event Tracing for Windows)** — вбудований у ядро Windows механізм високопродуктивного трасування подій, фундамент, на якому будується телеметрія Sysmon та більшості EDR-платформ. *(14.10)*

**GPO (Group Policy Object)** — контейнер налаштувань безпеки Windows, що прив'язується до організаційної одиниці Active Directory й автоматично застосовується до вхідних до неї комп'ютерів і користувачів. *(14.3)*

**JIT (Just-In-Time Access)** — модель тимчасового, обмеженого в часі надання привілейованого доступу; вперше розглянута в Модулі 05, тут пов'язана з Tiered Administration Model. *(14.11)*

**journald** — компонент systemd, що централізовано збирає системні журнали в структурованому бінарному форматі, доступний через `journalctl`. *(14.9)*

**LAPS (Local Administrator Password Solution)** — рішення Windows для автоматичної ротації унікальних локальних адміністративних паролів на кожній машині; вперше розглянуте в Модулі 05. *(14.4)*

**LSASS (Local Security Authority Subsystem Service)** — центральний процес Windows, що відповідає за автентифікацію користувачів і тимчасово зберігає їхні облікові дані в пам'яті; головна ціль credential dumping атак. *(14.4)*

**PAM (Pluggable Authentication Modules)** — модульна архітектура автентифікації Linux, що дозволяє гнучко налаштовувати механізм перевірки особи незалежно від конкретного застосунку. *(14.5)*

**PAW (Privileged Access Workstation)** — фізично чи логічно окрема, суворо захищена робоча станція, призначена виключно для адміністративних завдань певного рівня. *(14.11)*

**PPL (Protected Process Light)** — механізм Windows, що обмежує, які процеси можуть відкривати дескриптор доступу до LSASS для читання його пам'яті; технічна основа LSA Protection. *(14.4)*

**RSOP (Resultant Set of Policy)** — фактичний, підсумковий набір налаштувань Group Policy, що застосовується до конкретної машини чи користувача після об'єднання всіх застосовних GPO. *(14.3)*

**SELinux (Security-Enhanced Linux)** — реалізація MAC на основі міток типів (type enforcement), що визначає дозволену поведінку процесів на рівні доменів безпеки, незалежно від DAC-прав. *(14.6)*

**SIEM (Security Information and Event Management)** — платформа централізованої агрегації, кореляції та аналізу журналів безпеки з багатьох джерел, кінцева точка форвардингу для Sysmon, auditd, Windows Event Forwarding. *(14.9)*

**Sysmon (System Monitor)** — компонент Sysinternals, що доповнює стандартний Windows Event Log значно деталізованішою телеметрією, спеціально призначеною для виявлення підозрілої активності. *(14.8)*

**Tiered Administration Model** — модель ярусного адміністрування (Tier 0/1/2), що ізолює облікові дані різних рівнів привілеїв одне від одного, унеможливлюючи компрометацію найвищого рівня через найнижчий. *(14.11)*

**VBS** → Віртуалізована безпека.

**WEF (Windows Event Forwarding)** — вбудований механізм Windows для централізованого пересилання журналів подій з багатьох машин на єдиний колектор без стороннього агента. *(14.8)*

**WSUS (Windows Server Update Services)** — локальний сервер, що дозволяє централізовано схвалювати й розповсюджувати оновлення Microsoft для груп комп'ютерів організації. *(14.7)*
